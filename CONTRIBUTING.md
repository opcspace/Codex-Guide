# 贡献指南

感谢你帮助这套中文实践知识库保持准确、实用和易读。

本仓库遵循 [OPCspace 组织贡献指南](https://github.com/opcspace/.github/blob/main/CONTRIBUTING.md)。大范围结构调整请先创建 Issue，小型文档修正可以直接提交 PR。

## 适合提交的改动

- 修正失效链接、过期入口或事实错误；
- 基于官方新文档补充实践内容；
- 增加有明确输入、步骤和验收标准的案例；
- 改善导航、搜索、无障碍或移动端体验；
- 修正错别字和不清楚的表达。

## 内容要求

1. 产品事实必须附官方来源，第三方文章不能作为唯一依据。
2. 不大段复制官方原文；应摘要、重组并说明实践意义。
3. 把事实与建议分开，不把个人推导写成官方结论。
4. 涉及命令、权限、产品入口时写明核查日期。
5. 案例应说明目标、输入、约束、验证和失败处理。
6. 不提交密钥、账户信息、真实客户数据或未脱敏截图。

## 本地检查

```bash
npm ci
npm run check
npm run build
```

构建后打开 `docs/`，检查桌面与手机宽度、内部链接、代码块和页内目录。

## 内容文件

- 在 `content/` 中编辑 Markdown；
- 在 `nav.json` 登记新页面和阅读顺序；
- 每页需要 `title` 与 `description` frontmatter；
- 外部来源集中补充到 `content/reference/sources.md`；
- 不直接手改 `docs/`，它由构建脚本生成。

## Pull Request 清单

- [ ] 改动范围单一且标题明确
- [ ] 官方来源与核查日期已提供
- [ ] `npm run check` 通过
- [ ] `npm run build` 通过
- [ ] 没有敏感信息或大段版权内容
- [ ] 已检查生成页面

维护者可能会根据官方资料更新、合并或调整内容结构。
