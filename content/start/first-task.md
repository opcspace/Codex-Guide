# 第一个任务，要小而完整

OpenAI 的入门建议是先选择一个本地文件夹或 Git 仓库，再发起任务。第一次不要挑战“大重构”，选一个 15–30 分钟可验收的闭环：解释模块、修复单个 Bug、补一个测试、更新一页文档。

## 开始前

1. 选择正确的项目目录，确认里面没有不希望 AI 读取或修改的秘密文件。
2. 最好在 Git 仓库中工作，先运行 `git status`，知道现有改动属于谁。
3. 默认使用受限权限；只有明确需要且可恢复时才扩大权限。

## 第一条任务模板

```text
请先阅读这个仓库，不要修改文件。

目标：定位用户登录失败时没有显示错误提示的原因。
范围：只检查 src/auth 和相关测试。
请输出：
1. 调用链和根因；
2. 最小修改方案；
3. 计划运行的验证命令；
4. 任何需要我确认的风险。
```

确认分析后，再发第二条：

```text
按最小方案实现。不要改公共 API。
运行相关测试和 lint；若失败，区分“你的改动导致”与“原有失败”。
最后列出修改文件、验证结果和仍未解决的问题。
```

## 验收四问

- 改动是否严格落在约定范围？
- 差异里有没有意外删除、格式化或依赖升级？
- 测试是否真的运行，输出是否支持“已完成”？
- 是否有你必须亲自判断的产品或安全问题？

## 失败时怎么继续

不要只说“再试一次”。指出可观察的偏差：哪一处不符合、期望是什么、哪些内容必须保留。长对话开始跑偏时，开新任务并带上精简后的事实和约束，通常比继续修补上下文更可靠。
